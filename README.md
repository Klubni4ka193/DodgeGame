# Dodge Game

## Описание проекта

**Dodge Game** — это простая 3D-игра, разработанная в Unity. Игрок управляет шаром на ограниченной арене и должен как можно дольше уклоняться от падающих сверху кубов-препятствий.

Игра была разработана в рамках лабораторной работы для изучения базовых возможностей Unity: создания игровых объектов, работы с физикой, коллайдерами, prefab-объектами, пользовательским интерфейсом и C#-скриптами.

## Цель игры

Цель игрока — продержаться как можно дольше, избегая столкновения с падающими препятствиями.

Чем дольше игрок находится в игре, тем выше становится счёт. При столкновении с препятствием игра завершается, появляется надпись **Game Over** и кнопка перезапуска.

## Основные возможности

- управление игроком с клавиатуры;
- движение игрока по арене;
- случайное появление падающих препятствий;
- физическое падение кубов с использованием `Rigidbody`;
- определение столкновения игрока с препятствием;
- отображение счёта на экране;
- завершение игры при столкновении;
- кнопка перезапуска игры;
- постепенное повышение сложности игры.

## Управление

| Клавиша | Действие |
|---|---|
| W / ↑ | Движение вперёд |
| S / ↓ | Движение назад |
| A / ← | Движение влево |
| D / → | Движение вправо |

## Игровой процесс

1. После запуска игры игрок появляется на арене.
2. Сверху начинают падать кубы-препятствия.
3. Игрок должен уворачиваться от препятствий.
4. Счёт увеличивается каждую секунду.
5. Со временем препятствия начинают падать быстрее, а игра становится сложнее.
6. При столкновении с кубом игра завершается.
7. На экране появляется надпись **Game Over** и кнопка **Restart**.
8. При нажатии на кнопку **Restart** сцена перезапускается.

## Используемые компоненты Unity

В проекте использовались следующие компоненты Unity:

- `GameObject` — игровые объекты сцены;
- `Transform` — настройка позиции, поворота и масштаба объектов;
- `Rigidbody` — физика движения игрока и препятствий;
- `Collider` — обработка столкновений;
- `Prefab` — шаблон препятствия;
- `Canvas` — пользовательский интерфейс;
- `TextMeshPro` — отображение текста счёта и сообщения о проигрыше;
- `Button` — кнопка перезапуска игры;
- C#-скрипты — реализация логики игры.

## Описание основных скриптов

### PlayerController.cs

Скрипт отвечает за движение игрока и обработку столкновения с препятствием.

Основные функции:

- получение ввода с клавиатуры;
- применение силы к `Rigidbody` игрока;
- проверка столкновения с объектом, имеющим тег `Obstacle`;
- вызов завершения игры через `GameManager`.

### ObstacleSpawner.cs

Скрипт отвечает за создание падающих препятствий.

Основные функции:

- случайный выбор позиции появления куба;
- создание препятствий из prefab;
- настройка скорости падения препятствий;
- постепенное повышение сложности игры.

### Obstacle.cs

Скрипт отвечает за удаление препятствий спустя определённое время после появления.

Это нужно для того, чтобы старые кубы не накапливались в сцене и не снижали производительность игры.

### GameManager.cs

Скрипт управляет общим состоянием игры.

Основные функции:

- подсчёт очков;
- отображение счёта;
- обработка состояния проигрыша;
- отображение текста `Game Over`;
- отображение кнопки `Restart`;
- перезапуск сцены.

## Повышение сложности

В игре реализовано постепенное повышение сложности. Через заданные промежутки времени скорость падения препятствий увеличивается, а интервал между их появлением уменьшается.

Это делает игровой процесс более динамичным: чем дольше игрок выживает, тем сложнее становится уворачиваться от падающих кубов.

## Условие проигрыша

Игра завершается, если объект игрока сталкивается с препятствием.

Для проверки столкновения используется метод:

```csharp
private void OnCollisionEnter(Collision collision)
{
    if (collision.gameObject.CompareTag("Obstacle"))
    {
        gameManager.GameOver();
    }
}
```

Препятствия имеют тег `Obstacle`, благодаря чему скрипт может отличать их от других объектов сцены.

## Пользовательский интерфейс

В игре используется простой интерфейс:

- `ScoreText` — отображает текущий счёт игрока;
- `GameOverText` — появляется после проигрыша;
- `RestartButton` — позволяет перезапустить игру.

Интерфейс расположен на объекте `Canvas` в режиме `Screen Space - Overlay`.
